Ubicación: Dockerfile:1
Control Relacionado: ISO 27001 - Control de Acceso Privilegiado

El archivo Dockerfile no define una instrucción 'USER', haciendo que el contenedor corra por defecto como 'root'. Esto representa un vector de vulnerabilidad si un atacante logra escapar del proceso.

Sugerencia de Dockerfile Seguro 
FROM node:18-alpine

# Crear usuario seguro sin privilegios
RUN addgroup -S appgroup && adduser -S appuser -G appgroup

WORKDIR /app
COPY package*.json ./
RUN npm ci

COPY . .

# Usar el usuario seguro
USER appuser

EXPOSE 3000
CMD ["npm", "start"]
